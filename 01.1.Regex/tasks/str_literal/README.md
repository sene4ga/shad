## Числовой литерал

### Введение

См. аналогичный раздел в задании про шестнадцатиричный числовой литерал.


### Условие

Исходим из того, что у нас есть функция

```python
def extract_token(s, regexp, f_repr):
    if m := re.match(regexp, s):
        return f_repr(m), s[m.span()[1]:]
    else:
        return None, s
```

которая принимает:

- строку s

- регулярное выражение regexp

- функцию f_repr

Регулярное выражение должно привести в возвращению непустного Match-объекта, если строка s начинается
со строковой константы (правила формирования которой описаны ниже), которой может предшествовать пробельная
последовательность
 
Непустой Match-объект, должен возвращать span(), заканчивающийся ровно на последнем символе строковой константы
(закрывающей кавычки).

Решение должно состоять из:

- регулярного выражения, сохраненного в переменной RE_STR_LITERAL

- функции `f_repr_str_literal`, принимающей непустой Match-объект и возвращающей строку с представлением
строковой константы. Ведущие пробельные символы, идущие до открывающей кавычки не должны в нее попадать.

Строковая константа начинается с одинарной или двойной кавычки и заканчивается на символе кавычки того же
типа, с которого начиналась.

Между открывающей и закрывающей кавычками может 0, 1 или несколько описаний символов.

В качестве описания символа могут выступать:

- пара символов \n

- пара символов \t

- пара символов \\

- пара символов \' (только если строка начинается и заканчивается на одинарную кавычку)

- пара символов \" (только если строка начинается и заканчивается на двойную кавычку)
  
- " (только если строка начинается и заканчивается на одинарную кавычку)

- ' (только если строка начинается и заканчивается на двойную кавычку)

- любые символы, кроме табуляции, перевода строки, одинарных кавычек, двойных кавычек, \

Отдельно отметим, что \ может выступать только в сочетании с указанными символами. Все остальные сочетания
считаются незаконными.


### Примеры

'  "abc"123' - должен получиться непустой Match-объект и `extract_token` должна вернуть
пару ('"abc"', '123')

'  "x\"y' - extract_token` должна вернуть пару (None, '  "x\"y')
