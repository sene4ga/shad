## Числовой литерал


### Условие

Исходим из того, что у нас есть функция

```python
def find_all(s, regexp):
    return [e.group(0) for e in re.finditer(regexp, s)]
```

которая принимает:

- строку s

- регулярное выражение regexp

Регулярное выражение должно быть написано таким образом, чтобы приведенный вызов вернул список
целых чисел-палиндромов длиной не более, чем 5.

Палиндром - это строка, которая равна собственной инверсии (например, 'abba', 'несунгнусен')

Число-палиндром - это число, которое будучи воспринимаемо как строка, является палиндромом.

Если строже и точнее, то нас интересуют последовательности символов '0'-'9' длиной не выше, чем
5, ограниченные пробелами и/или границами строки и являющиеся палиндромами. При наличии ведующих
нулей мы при определении палиндрома их учитываем как есть.

### Примеры

'123  aba  121  1+1 9999' - должны вернуть ['121', '9999']

'02320 1 +1 09999' - должны вернуть ['02320', '1']
